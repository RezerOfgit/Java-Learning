你需要开发一个“部门管理”页面。产品经理要求：页面刚打开、表格刚渲染出来的时候， 里面就必须已经有后端的真实部门数据了。 提问：你写的那个 axios.get('/api/depts') 代码，必须放在 Vue 的哪个生命周期钩子（函数）里面？ 为什么? 必须放在 mounted 生命周期钩子里面（也可以放在 created，但更推荐 mounted）。 原因： mounted 在组件挂载到 DOM 后立即执行，此时页面已经完成初始渲染，但尚未填充数据。 在这个钩子中发起请求，可以确保表格容器已经存在，同时数据返回后通过响应式更新自动渲染到表格中， 用户看到的就是“打开即填充好数据”的效果。如果放在 beforeCreate 或 created 中虽然也能请求数据， 但无法保证 DOM 元素已准备就绪（例如需要对表格做第三方初始化），而 mounted 是兼顾“数据获取”与 “DOM 可操作”的最常用位置。
